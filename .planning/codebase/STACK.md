# Technology Stack

**Analysis Date:** 2026-03-26

## Languages

**Primary:**
- C++ 17 - Core application logic, UI, hardware control, algorithms (CONFIG += c++17 in `machineDecoupeIHM.pro`)

**Secondary:**
- Bash - Build scripts and system utilities (`scripts/setup_dev_ubuntu.sh`)

## Runtime

**Environment:**
- Raspberry Pi (primary target) with Linux kernel 6.8.0
- Ubuntu/Debian-based systems (development and production)
- Windows support for development (OpenCV build paths in `machineDecoupeIHM.pro` lines 14-23)

**Package Manager:**
- APT (Debian/Ubuntu) - System package management
- qmake6 - Qt project build system (not cmake)

## Frameworks

**Core GUI:**
- Qt6 (6.x) - Primary GUI framework with modules:
  - Qt6 Widgets - Window management and standard UI components
  - Qt6 SVG - Vector graphics rendering
  - Qt6 Network - HTTP and Bluetooth networking
  - Qt6 Bluetooth - Bluetooth connectivity (QT += bluetooth in `machineDecoupeIHM.pro` line 1)
  - Qt6 HttpServer - HTTP server for network file transfer
  - Qt6 OpenGL Widgets - 3D graphics rendering support

**Build:**
- qmake6 - Qt project configuration and build generation (primary build tool)
- make (GNU) - Build execution via Makefile generated by qmake6

**Testing:**
- Qt Test Framework - Unit testing via `tests/tests.pro`

## Key Dependencies

**Critical (system libraries via pkg-config):**
- OpenCV 4 (libopencv-dev) - Image processing for AI image generation, logo importing, edge detection
  - Used in: `infrastructure/imaging/` for skeletonization, image edge import
  - Linked on Linux via PKGCONFIG in `machineDecoupeIHM.pro` line 10
  - Manual linking on Windows (lines 14-23)

- libgpiod - GPIO control for Raspberry Pi hardware (motor control, pins)
  - Used in: `infrastructure/hardware/Raspberry.h`
  - Provides hardware interface to GPIO lines via gpiod_chip and gpiod_line APIs
  - Linked on Linux via PKGCONFIG in `machineDecoupeIHM.pro` line 10

**GUI Development:**
- qt6-base-dev, qt6-base-dev-tools - Qt6 core libraries and tools
- qt6-svg-dev - SVG support module
- qt6-connectivity-dev - Bluetooth and network connectivity
- qt6-httpserver-dev - HTTP server module
- libqt6opengl6-dev - OpenGL rendering
- linguist-qt6 - Translation tools (for bilingual FR/EN support)
- qt6-l10n-tools - Qt localization tools

**Build Toolchain:**
- g++ - C++ compiler
- gcc - C compiler
- make - Build execution
- cmake, ninja-build - Alternative build tools (installed but qmake6 is primary)
- pkg-config - Library configuration discovery
- build-essential - GCC and associated build tools

**Graphics/Display:**
- libgl1-mesa-dev - OpenGL implementation
- libegl1-mesa-dev - EGL windowing system for OpenGL

## Configuration

**Build Configuration Files:**
- `machineDecoupeIHM.pro` - Main qmake project configuration defining:
  - Qt modules (core, gui, widgets, svg, network, bluetooth, httpserver, openglwidgets)
  - C++ standard (c++17)
  - Platform-specific settings (Unix/Linux and Windows)
  - Include paths for all source directories
  - Header and source file listings
  - Resource files (qrc)
  - Translation files (ts)

- `tests/tests.pro` - Separate test project configuration

**Environment Configuration:**
- `OPENAI_API_KEY` - Required environment variable for OpenAI image generation
  - Read in: `infrastructure/network/OpenAIService.cpp` line 56
  - If missing, generation fails with message "La clé API OpenAI est absente"

- Platform-dependent paths:
  - Linux/Raspberry Pi: Uses pkg-config to locate OpenCV4 and libgpiod
  - Windows: Hard-coded paths to C:/opencv/build/ for OpenCV headers and libraries

**Translation Configuration:**
- `translations/machineDecoupeIHM_fr_FR.ts` - French localization
- `translations/machineDecoupeIHM_en_US.ts` - English localization
- Language enum in `shared/Language.h` controls runtime language selection

**Resource Files:**
- `resources.qrc` - Qt resource collection (images, icons, stylesheets)

## Platform Requirements

**Development:**
- Ubuntu/Debian with apt package manager
- qmake6 version 6.x or compatible
- g++ with C++17 support
- OpenCV 4 development headers
- libgpiod development headers
- Qt6 development packages (listed above)
- pkg-config for library discovery
- ~2-3 GB for Qt6 development environment

**Production (Raspberry Pi):**
- Raspberry Pi with Linux kernel (tested on 6.8.0)
- Qt6 runtime libraries (smaller footprint than dev packages)
- OpenCV 4 runtime libraries
- libgpiod runtime library
- GPIO hardware access (requires root or appropriate permissions)
- Network connectivity (Ethernet/WiFi via nmcli)
- Bluetooth hardware for wireless file transfer
- Touchscreen or mouse input

**Windows Development:**
- Visual Studio or compatible C++ compiler
- Manual OpenCV 4 installation (build included in repo at C:/opencv/)
- Qt6 for Windows

## Build and Execution

**Build from Source:**
```bash
./scripts/setup_dev_ubuntu.sh          # One-time environment setup
qmake6 machineDecoupeIHM.pro           # Generate Makefile
make -j$(nproc)                        # Compile (parallel)
```

**Build Tests:**
```bash
cd tests/
qmake6 tests.pro
make -j$(nproc)
./tests                                # Run test binary
```

**Target Application:**
- Executable: `machineDecoupeIHM` (defined as TARGET in .pro file)
- Run on Raspberry Pi: `./machineDecoupeIHM` (fullscreen on boot)
- Entry point: `main.cpp` - Creates QApplication, initializes WorkspaceViewModel, displays MainWindow fullscreen
